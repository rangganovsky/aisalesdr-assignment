# Engineering Lead Assessment - Take Home Assignment

## Scenario

AI Sales Doctor helps B2B companies find and qualify leads using AI, 3rd party databases and enrichment APIs.

Your task is to build a **simple Lead Management MVP** that:
1. Displays leads
2. Allows adding new leads
3. Supports basic filtering
4. Describe how you scale this system

### Part A – Backend (25 min)
Build a simple API using **Python (FastAPI)**.
You may store data **in memory** (no database required).

### Part B – Frontend (25 min)
Use any **React.js** framework.
**UI Requirements**
* [Page 1] List leads in a table with id, name, job title, phone number, company, email, headcount and industry.
* [Page 2] Form to add new lead(s)
* Filter by industry and headcount
* Button to “Enrich” lead(s)

### Part C – Engineering Lead Thinking (10 min)
**1. Architecture & Scaling**
How would you scale this system to:
* 1M+ leads
* Multiple enrichment providers
* CRM integrations
* Campaign automation

**2. Team & Process**
If you joined AI Sales Doctor:
* First 90-day engineering priorities
* Team structure (who to hire first) , assuming you are the only engineer now
* Dev process (CI/CD, reviews, releases)

**3. Tradeoffs**
What shortcuts did you take due to time?
